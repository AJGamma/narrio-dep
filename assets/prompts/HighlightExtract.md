# Highlight Extract

从长文本中提取 3-5 个最引人注目的句子作为亮点，用于预览或封面展示。

## 输出格式

使用 JSON 格式，包含 highlights 数组：

```json
{
  "highlights": [
    {
      "rationale": "为什么这个句子吸引人",
      "text": "提取的句子原文",
      "score": 0.95,
      "position": {
        "sentence_index": 12,
        "total_sentences": 150
      }
    }
  ]
}
```

## 评分标准

分数范围 0-1，根据以下维度评估：

### 高分数特征 (0.8-1.0)
- **情感共鸣**：能引发读者情感反应
- **惊喜因子**：反直觉、意外的观点或事实
- **可引用性**：独立成句，适合直接引用
- **洞察密度**：一句话传达深刻见解
- **自包含性**：脱离上下文仍能理解

### 低分数特征 (0-0.5)
- 依赖代词指代（这个、那个、它等）
- 需要前后文才能理解
- 过于抽象或泛泛而谈
- 包含行话或专业术语
- 平淡的陈述句

## 文章类内容特别关注

- 洞察密度高的句子
- 反直觉的观点和发现
- 主题句和核心论点
- 具有启发性的总结
- 打破常规认知的陈述

## 播客类内容特别关注

- 对话中的金句
- 情感真挚的时刻
- 自然口语化的精彩表达
- 有叙事吸引力的片段
- 嘉宾的独特观点

## 避免选取

- 开头第一句和最后一句（通常是引入和总结）
- 包含"如上所述""如下文将述"等过渡语的句子
- 高度依赖上下文的代词句
- 纯粹的举例说明
- 过于技术性的细节描述

## 要求

1. 返回 3-5 个亮点，按 engagement score 降序排列
2. 每个亮点必须是完整的句子
3. 确保亮点脱离原文后仍可理解
4. 为每个选择提供简短的 rationale 说明
5. 优先选择能激发好奇心的内容
